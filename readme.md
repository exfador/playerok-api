## 🤖 **Возможности**

### 🛒 **Playerok и автоматизация**

* **Автовыдача** — ключевые фразы в названии лота → текст или пакет файлов при продаже (`conf/auto_deliveries.json`).
* **Автоподнятие** — подъём лотов по расписанию, белые/чёрные списки по названию (`conf/auto_bump_items.json`).
* **Автовосстановление** — возврат проданных/истёкших лотов на витрину (`conf/auto_restore_items.json`, `auto.restore`).
* **Автоподтверждение сделок** — по ключевым фразам в названии (`conf/auto_complete_deals.json`).
* **Пользовательские команды** — триггеры в чате на площадке (`conf/custom_commands.json`).
* **Ватермарк** — подпись к исходящим сообщениям на Playerok (`features.watermark`).
* Работа с сайтом по **HTTP + WebSocket**, JWT-сессия и User-Agent.

### 🗨️ **Уведомления и панель в Telegram**

* Уведомления о **чатах**, **сделках**, **отзывах**, жалобах и системных событиях — с гибкими переключателями (`alerts.on`).
* **Ответ покупателю** прямо из Telegram — текст или фото.
* **Шаблоны** с переменными (`$buyer`, `$seller` и др.), история чата постранично, ссылка на чат на сайте.
* Из уведомления по сделке: ответ, шаблоны, **закрытие** и **возврат**.
* **Профиль** — баланс, активные лоты и сделки.
* **Статистика сессии** — закрытые сделки, возвраты, заработок; сброс из настроек.
* **Пароль панели** (хэш **SHA-256**); после входа ваш `user id` попадает в список админов.
* **Прокси** отдельно для Playerok и для Telegram API (в т.ч. **SOCKS5** `socks5h:…`), проверка при первом запуске.

### ⚙️ **Дополнительно**

* **Расширения** — модули в `ext/`: хуки шины событий и роутеры aiogram.
* **Служебные команды:** `/start`, `/stats`, `/logs`, `/restart` и `/reboot`.
* Логи `logs/bot.log` с лимитом размера; данные в `conf/` и `db/` (JSON).

## ⚙️ **Конфигурация**

Главный файл: **`conf/config.json`** (недостающие поля подтягиваются из дефолтов при запуске).

| Поле / раздел | Описание |
|---------------|----------|
| `bot.token` | токен от @BotFather |
| `bot.password_hash` | SHA-256 пароля (задаётся в мастере первого запуска) |
| `bot.proxy` | прокси для Telegram API |
| `bot.admins` | список Telegram user id с доступом к панели |
| `account.token` | JWT Playerok (cookie `token`) |
| `account.user_agent` | User-Agent браузера |
| `account.proxy` | прокси для запросов к Playerok |
| `account.timeout` | таймаут запросов, сек. |
| `features` | ватермарк, приветствие, чат, команды, автовыдача… |
| `auto` | восстановление, подтверждение, бамп |
| `alerts` | какие уведомления слать |
| `logs.max_mb` | максимум размера лог-файла |
| `debug.verbose` | подробный лог |

Также: `conf/messages.json`, `conf/custom_commands.json`, `conf/auto_deliveries.json`, `conf/auto_restore_items.json`, `conf/auto_complete_deals.json`, `conf/auto_bump_items.json`.

## ⬇️ **Установка и запуск**

1. Зависимости:

```bash
pip install -r requirements.txt
```

2. Запуск из корня репозитория:

```bash
python main.py
```

3. **Первый запуск** — мастер в консоли спросит токен бота, пароль панели, при необходимости прокси, JWT Playerok и User-Agent, затем проверит доступность Playerok и Telegram.

4. В Telegram: `/start` → ввод пароля → главное меню. Дальше всё настраивается из **«Настройки»** в боте.
