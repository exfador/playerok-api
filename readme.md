# CXH Playerok

**Telegram-панель для продавцов на маркетплейсе [Playerok](https://playerok.com):** уведомления о чатах, сделках и отзывах, ответы покупателю из Telegram (текст и фото), автовыдача по ключевым фразам в названии лота, автоподнятие и возврат лотов на витрину, автоподтверждение сделок по правилам, шаблоны сообщений и пользовательские команды в чате площадки. Движок опрашивает ленту событий Playerok и синхронизируется с сайтом через HTTP/WebSocket; интерфейс управления — **aiogram 3**.

Локальные данные: JSON-конфиги в `conf/`, состояние в `db/` (пользователи, статистика, сохранённые лоты, метки времени событий). Логи — `logs/bot.log` с ограничением размера.

---

## Возможности

- **Уведомления в Telegram** — новые сообщения в чатах, системные события по сделкам, смена статуса, жалобы и их снятие, отзывы; отдельные переключатели в `alerts.on` (сообщения, сделки, отзывы, проблемы, восстановление лотов, бамп и др.).
- **Чаты** — переход в чат на сайте, просмотр истории (постранично), ответ из бота текстом или фото, быстрые **шаблоны** с подстановкой переменных (`$buyer`, `$seller` и т.д.).
- **Сделки** — уведомления о новых и изменённых сделках; из уведомления: ответ, шаблоны, закрытие сделки, возврат.
- **Отзывы** — уведомления; переход к ответу на отзыв или написание в чат.
- **Автовыдача** — правила в `conf/auto_deliveries.json`: ключевые фразы в названии товара → выдача текста или «пакета» файлов при продаже.
- **Автовосстановление** — возврат проданных/истёкших лотов на полку по расписанию и спискам включений (`conf/auto_restore_items.json`, блок `auto.restore` в конфиге).
- **Автоподнятие** — подъём позиций лотов по интервалу; белые/чёрные списки по фразам в названии (`conf/auto_bump_items.json`, `auto.bump`).
- **Автоподтверждение сделок** — закрытие по ключевым фразам в названии (`conf/auto_complete_deals.json`, `auto.confirm`).
- **Пользовательские команды** — триггеры в чате Playerok (`conf/custom_commands.json`), ответы из настроек.
- **Ватермарк** — префикс/суффикс к исходящим сообщениям на площадке (`features.watermark`).
- **Профиль и баланс** — сводка по аккаунту Playerok из панели (баланс, активные лоты и сделки).
- **Статистика сессии** — счётчики закрытых сделок, возвратов, условного заработка; сброс из настроек.
- **Расширения** — Python-модули в каталоге `ext/`: хуки системы/рынка, роутеры Telegram, метаданные плагина.
- **Безопасность входа** — пароль панели (хэш **SHA-256**); при верном пароле `telegram user id` добавляется в список администраторов.
- **Прокси** — отдельно для запросов к Playerok и для Telegram API (в т.ч. SOCKS5 `socks5h:…`); проверка при первом запуске.
- **Служебные команды:** `/start` — меню, `/stats` — статистика, `/logs` — работа с логами, `/restart` и `/reboot` — перезапуск процесса.

## Конфигурация

Основной файл: **`conf/config.json`** (структура дополняется при обновлении дефолтов приложения).

| Поле / раздел | Описание |
|---------------|----------|
| `bot.token` | токен бота от @BotFather |
| `bot.password_hash` | SHA-256 пароля панели (задаётся при первом запуске в консоли) |
| `bot.proxy` | прокси для Telegram API (часто нужен в РФ) |
| `bot.admins` | список `user id` Telegram с доступом к панели |
| `account.token` | JWT сессии Playerok (cookie `token`) |
| `account.user_agent` | User-Agent браузера для запросов к сайту |
| `account.proxy` | прокси для Playerok |
| `account.timeout` | таймаут HTTP-запросов, сек. |
| `features` | ватермарк, приветствие, чтение чата, команды, автовыдача и др. |
| `auto` | автовосстановление, автоподтверждение, автоподнятие |
| `alerts` | вкл/выкл типов уведомлений |
| `logs.max_mb` | лимит размера `logs/bot.log` |
| `debug.verbose` | подробные логи |

Дополнительные JSON: `conf/messages.json` (шаблоны), `conf/custom_commands.json`, `conf/auto_deliveries.json`, `conf/auto_restore_items.json`, `conf/auto_complete_deals.json`, `conf/auto_bump_items.json`.

## Установка и запуск

1. Установите зависимости:

```bash
pip install -r requirements.txt
```

2. Запустите из корня проекта:

```bash
python main.py
```

3. При **первом запуске** в консоли по шагам запросятся: токен Telegram-бота, пароль панели, при необходимости прокси для Telegram и для Playerok, JWT Playerok, User-Agent. После ввода выполняется проверка доступности Playerok и Telegram API.

4. В Telegram откройте бота, выполните `/start`, введите пароль — вы попадёте в главное меню. Дальнейшая смена токена, прокси и параметров — через раздел **«Настройки»** в боте.

## Расширения
